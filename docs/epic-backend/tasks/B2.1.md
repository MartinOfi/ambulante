# B2.1 — Policies RLS de todas las tablas del dominio

> **Fase:** B2 — RLS policies + pgTAP tests + performance RLS
> **Goal de la fase:** Activar Row Level Security en todas las tablas del dominio, con tests automatizados que garantizan tanto la corrección como la performance.
> **Acceptance criteria de la fase:** Cada tabla tiene `enable row level security` + `force row level security` + policies por rol (anon, authenticated, service_role); suite pgTAP cubre casos positivos/negativos por rol; benchmark RLS valida <20ms para queries críticas con 10k stores / 100k orders.
