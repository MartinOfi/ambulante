# B7.4 — Runbook: cómo desactivar / reactivar / auditar un cron

> **Fase:** B7 — Cron & system jobs (pg_cron + pg_net + state machine)
> **Goal de la fase:** Dejar los timeouts del PRD §7.6 automatizados — `EXPIRADO` a 10min sin respuesta, auto-close a 2h de ACEPTADO — usando pg_cron que dispara Route Handlers Next.js con la state machine TS.
> **Acceptance criteria de la fase:** pg_cron tiene 2 schedules activos; los Route Handlers usan `SKIP LOCKED` para soportar workers concurrentes; tests de integración verifican correctness bajo concurrencia.
